# 项目规则索引

本项目采用分层管理的规则体系，主文件作为索引，详细规则分布在 `sections/` 目录下的子文件中。

## 规则文件结构

### 1. 核心规则

- **文件路径**: `sections/core_rules.md`
- **内容**: 项目框架版本、依赖要求和禁止使用的 API

### 2. 文件结构规则

- **文件路径**: `sections/file_structure.md`
- **内容**: 文件命名规范、单一职责原则、功能示例结构和导出规范

### 3. 测试规则

- **文件路径**: `sections/testing_rules.md`
- **内容**: 测试框架要求和测试文件更新规范

### 4. 文档规则

- **文件路径**: `sections/documentation_rules.md`
- **内容**: 文档更新规范和注释规范

## 使用说明

1. 所有开发者应熟悉并遵循所有规则
2. 规则变更应通过 PR 进行，确保团队共识
3. 新增功能或修改现有功能时，应检查并更新相关规则
4. 规则文件应保持简洁、清晰，便于查阅和执行

## 规则版本

- **版本**: 1.0.0
- **更新日期**: 2026-01-15
- **更新说明**: 初始版本，采用分层管理结构
